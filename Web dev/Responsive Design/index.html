Raccoglierò qui tutte le informazioni e i consigli utili per creare una pagina web responsive. Potrebbe essere soggetta ad aggiornamenti futuri.

Breakpoint

I breakpoint sono servono a dire che regole CSS usare in base a delle caratteristiche specifiche (larghezza del dispositivo per esempio). Io solitamente uso quelli di Bootstrap e da quanto ho capito non è necessario creare dei @media query per supportare le risoluzioni più grandi, spesso i contenuti vengono zoommati.

@media (max-width: 575.98px) { /* Regole */ } @media (max-width: 767.98px) { /* Regole */ } @media (max-width: 991.98px) { /* Regole */ } @media (max-width: 1199.98px) { /* Regole */ } @media (max-width: 1399.98px) { /* Regole */ }

Immagini

È inutile far caricare immagini in 4k su uno smartphone, bastano delle immagini a media-bassa risoluzione, per fare lo swap di immagini automaticamente puoi il tag picture.

<picture> <source media="(min-width:650px)" srcset="img_pink_flowers.jpg"> <source media="(min-width:465px)" srcset="img_white_flower.jpg"> <img src="img_orange_flowers.jpg" alt="Flowers" style="width:auto;"> <picture>

Attenzione ai display a retina (es. telefoni)! Nei telefoni spesso un pixel corrisponde a due o più pixel di un normale display e questo vuol dire che se sul pc va bene un'immagine 500x250 su un telefono 2x serve una risoluzione di 1000x500. La scelta migliore è quella di usare immagini vettoriali ma se non si può puoi sfruttare il tag img e l'attributo srcset:

<img src='illustration-small.png'> srcset='images/illustration-small.png 1x, images/illustration-big.png 2x' <style='max-width: 500px' />

oppure

<img src='images/photo-small.jpg'> srcset='images/photo-big.jpg 2000w, images/photo-small.jpg 1000w' sizes='(min-width: 960px) 960px, <100vw' />

In questo caso per "2000w" si intende la dimensione in pixel della larghezza dello schermo.

Meta tag

Sarà chiaro, non ho capito molto di questo meta tag, le uniche cose che ho capito è che non è una cosa approvata da W3, serve per bloccare lo zoom nei dispositivi mobile ed evita problemi di responsive, tu inseriscilo sempre in caso.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Se non è fluido va anche bene

Hai presente i siti potenti in cui tutto è "fluido"? Bene, fregatene se il tuo sito non è così, la cosa principale è che si veda bene ovunque.

Only screen...

Alcune volte puoi trovare scritto @media screen and (max-width:632px) oppure @media only screen and (max-width:632px) e vuol dire che quel media query si attiverà quando si utilizza (solo) uno schermo che deve avere una larghezza massima di 632px ma se vuoi creare una pagina personalizzata quando si prova a stampare quest'ultima allora devi usare "print" al posto o assieme a screen. Per saperne di più guarda la documentazione ufficiale.
